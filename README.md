🏗️ Architect's Journey: Code Failure Dictionary
This repository serves as a "Steel-Standard" reference for Python development, built with the philosophy of "Learn by Failing." Instead of just documenting successful code, this project deep-dives into the mechanical limits of Python by intentionally triggering and documenting errors.

🛠️ Project Philosophy
Monozukuri (Artisanship): Approaching software engineering with a focus on technical perfection and deep-niche (Tokushu) understanding.

Sovereign Infrastructure: Building knowledge that enables local, cost-effective AI deployment without reliance on expensive APIs.

Anti-Vibe Coding: Transitioning from "guessing" code to understanding the exact "Species" (Types) and "Grammar" (Syntax) of Python.

📚 Error Index
The repository is structured into specific "Phase-wise" files to prevent memory overload and ensure clarity:
Phase,File,Focus
01,name_error.py,Identity crises and undefined variables.
02,syntax_error.py,Violations of Python's structural laws.
03,type_error.py,"Conflicts between different data ""Species""."
04,index_error.py,Boundary breaches in strings and sequences.
05,attribute_error.py,"Using the wrong ""Tool"" for a data type."
06,indentation_error.py,Physical alignment and structural integrity.
07,value_error.py,Incompatible content within a correct data type.
🚀 Deployment Instructions
To clone and use this dictionary in your local environment:
# Clone the repository
git clone https://github.com/FazalAbbas0/Architect_Journey.git

# Move into the breakage directory
cd breakage_test_codes

# Run a specific error test
python name_error.py
